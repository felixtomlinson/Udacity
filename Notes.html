General Notes

Container elements: elements that do nothing but contain stuff inside themselves
span div control how text laid out
Most pages have a doctype, head, body
Main focus is the body (CSS and Java Script outside this)
Indentation important
People need to read code too.

HTML CSS and Javascript all have different purposes

HTML - structure
CSS - style
Javascript - interactive elements

Tree like structure
HTML - language, tags turned into elements forming a tree
Document object model - standard convention for representing and interacting with html
CSS allows you to say all X with class should look like Y
Once you know language, you will have idea as to what it will look like.
Everything is a box
Thinking in terms of boxes makes your life as front end web developer much easier
Grid helps to make sure everything is aligned
Start with the big boces and move down from there
Classes are labels on boxes - remind you. Can belong to multiple classes
Avoid repetition
CSS stands fro cascading style sheets
Several sheets can be references from same HMTL.
Most specific rules applied. Go through elements until you find most specific one.
CSS write selector, select HTML tag and apply to all tags
p { } would apply to all paragraphs
apply to a class .X {X:X;}
Learn more about colours and how to define them? More specific than generic colours. CSS has a lot of functionality.
Call lots of different types of colours and fonts.
Everything betwee { } is a rule
The line of code that says background-color : black; is a declaration. background-color is a property and black is the value of that property.
Use Classes to differentiate different styles for different parts of the document
Can write CSS at the top of your HTML document
Can link to CSS in a separate file
Can write your style inline in html
Semantic conventions often associated with styles
There are default style types
Box model - margins, borders, padding, content.
Content - image or text. Nucleus (protected by other things, don't overlap or look to close)
Padding - green colour, clears an area around the content, affected by background colour of box
Border - around padding and content - colour of box
Margin - round the box - totally transparent, space around boxes.
Can play around with width of all of these things.
Size of element equal to whole thing.
Difficult to know how much apace box will take.
*{box-sizing: border-box;} includes border and boxes - size of box will always stay the same. Margin not included.
Drawback if you want to make sure older browsers can display it - browser specific instructions ahead of border-box
e.g. -webkit-box-etc., -moz-etc., -ms-etc.
Set size as pixel or percentage. Percentage changes to screen size.
Max width
Positioning complicated
Flexbox
Parent container to value flex : element.style {display: flex;}
